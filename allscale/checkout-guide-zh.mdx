## 1. 用户支付体验

### 1.1 进入收银台

用户在商户的 APP 或网站完成选品后，选择通过 AllScale 进行稳定币支付。商户系统调用 API 创建 Checkout Intent 后，用户会被引导跳转至 AllScale 收银台页面。

收银台页面顶部会展示商户的 Logo 和店铺名称，让用户清楚知道自己在为哪家商户付款。页面中间展示订单信息，包括订单描述、订单号和支付金额。

---

### 1.2 支付方式

收银台目前支持两种支付方式：

#### AllScale Pay

用户登录 AllScale 账户后，可直接使用钱包内的稳定币余额完成支付。用户选择余额充足的链，确认金额和 Gas 费后即可完成付款，全程无需离开收银台页面。


#### 插件钱包

用户也可以选择通过 OKX Wallet 或 MetaMask 等插件钱包完成支付。连接钱包后，在钱包 APP 内确认交易，支付完成后自动返回收银台。


---

### 1.3 支付完成

交易上链确认后，收银台显示支付成功页面，展示交易详情。用户可一键跳转回商户配置的指定页面，完成整个购买流程。

---

## 2. 商户后台功能

### 2.1 Store 设置

**店铺信息**
登录后台后，商户可在 Store 设置页面配置店铺名称（最多 64 个字符），这些信息会直接展示在用户的收银台页面上。

**API Key**
联系 BD，获得你的 API Key 和 API Secret。后台展示商户的 API Key，支持一键复制。API Secret 展示后四位，用于开发接入时的身份验证。

**Webhook 配置**
商户可在此配置 Webhook URL，用于接收支付结果的实时推送通知。


---

### 2.2 订单管理与对账

可在后台查看所有 Checkout Intent 订单，列表默认按时间降序排列，展示订单 ID、金额、发起时间和当前状态，支持按订单状态进行筛选。

**实时邮件通知**
每笔订单发生状态变化时，商户会收到对应的邮件通知：
- **支付成功：** 包含订单号、金额、法币折算金额、链上交易哈希、网络信息
- **支付失败（极低概率）：** 包含订单号、失败原因，提示用户可重新发起支付

**每日 / 每月汇总报告**
每天及每月 1 日 UTC 0:00 触发（有订单状态更新时发送），包含总交易笔数、成功 / 失败 / 待支付笔数，以及可下载的 CSV 明细报告。
